# Agentの自然言語を多言語化する

開発、仕様、テストなど、Sub-Agentを分けるような単位で指示言語を分ける。

同じプロジェクト内部で、ドキュメント化にはドキュメント用の指示で自動化し、ドキュメント単位のC3L/Climptしか話さないようにする。

Issue化するAgentは、そもそも言語が異なるので、Sub-Agent設定は不要。
大元の settings.json で分ける。作業完了のタイミングや、自動で動く時間軸も異なる。1時間に1回、現状に対して状況確認を入れて問題がある箇所をIssue化・Issue整理する。


開発担当のAgentは、コーディングのC3L/Climptしか話さないようにする。まだ実現していないが、バックエンドとフロントエンドも完全に分離できる見込みである。テスト先任者に分けることも可能だ。

彼らAgentは、常にIssueに基づいて修正処理を行い続ける。 *-drivenなど関係なく、全ての業務に特化したそれぞれの専門言葉をプロジェクトに特化して話す。
